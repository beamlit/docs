Log streaming provides developers with visibility into process outputs within sandboxes.

<Tip>Complete code examples demonstrating all operations are available on Blaxel's GitHub: [in TypeScript](https://github.com/beamlit/sdk-typescript/blob/main/integrationtest/sandbox.ts) and [in Python](https://github.com/beamlit/sdk-python/blob/main/integrationtest/sandbox.py).</Tip>

## Retrieve logs

### Batch

This allows you to retrieve logs for a specific [process](Processes) (using either its name or process ID) after it has completed execution. This is useful for analyzing process output or debugging issues:

<CodeGroup>

```typescript TypeScript {6}

import { SandboxInstance } from "@blaxel/sdk";

const sandbox = await SandboxInstance.get("my-sandbox")
const process = await sandbox.process.exec({name: "test", command: "echo 'Hello world'"})

const logs = await sandbox.process.logs("test");

```

```python Python {15}

import asyncio
import logging
import os

from blaxel.client.models import Metadata, Port, Runtime, Sandbox, SandboxSpec
from blaxel.sandbox.base import ResponseError
from blaxel.sandbox.client.models import ProcessRequest
from blaxel.sandbox.sandbox import SandboxInstance

logger = logging.getLogger(__name__)

sandbox = await SandboxInstance.get(sandbox_name)
process = await sandbox.process.exec(ProcessRequest(name="test", command="echo 'Hello world'"))

logs = await sandbox.process.logs("test")

```
